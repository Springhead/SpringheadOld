\section{ルーチン作業}
\label{sec:RoutineWork}

\noindent
DailyBuildの結果をWikiの\KQuote{開発者向け情報}のページで確認する。\\
URL:\Hskip{1em}\tt{http://springhead.info/wiki/index.php?devel}

\begin{enumerate}
  \item	「開発版」にある次の2つのリンクをたどり、
  	以前の結果と相違がないか確認する。
	\begin{itemize}
	  \item	今朝のビルド − 今朝のビルドの状況(\UL{\bf{過去のビルドの履歴}})
	  \item	今朝のビルド − Samplesのビルドの状況(\UL{\bf{過去のビルドの履歴}})
	\end{itemize}

  \item	相違があったときは``今朝のビルドのログ''等を見て原因を特定する。
	\begin{itemize}
	  \item	テスト系の不具合のときは修正をする。
	  \item	開発系の不具合のときは開発者に連絡をし、必要なら対処を依頼する。
		\begin{narrow}
			メール先： springhead開発者 \tt{<dev@springhead.info>}
		\end{narrow}
		さらにGitHubのissuesへ登録も検討する。
		\begin{narrow}
			リンク： 「基本情報」− \UL{\bf{バグトラッカ}}
		\end{narrow}
		\begin{description}
		  \item[※]
			今朝のビルドが最後まで実行できていない場合には、
			DailyBuildを実行するマシン
			\tt{DESKTOP-KD3C7HS}の
			\begin{narrow}
				\Path{I:/DailyBuild/DailyBuild.log}\\
				\Path{I:/DailyBuild/MakeReport.log}
			\end{narrow}
			を見て原因を特定する必要がある。
		\end{description}
	\end{itemize}

  \item	週に1回程度は次も確認する。
	\begin{enumerate}
	  \item	ドキュメントページからAPIリファレンス のリンクをたどり、
		ページが正しく作成されているか確認する。
		特に、ヘッダファイルのインクルード依存関係図が
		生成されているか確認する
		(例えば、
		\LSB ユーティリティクラス\RSB --\LSB\tt{BaseUtility.h}\RSB%
		とたどってみる)。
	  \item	ドキュメントページからWindows Helpファイルをダウンロードし、
		上記と同様に内容を確認する。
		ダウンロードしたファイル(\tt{Springhead.chm})は、
		プロパティウィンドウを開き、
		セキュリティフィールドの「ブロックの解除」を行なわないと
		内容が表示されないので注意すること。

		\LSB\bf{注意}\RSB 現在Helpファイルは作成できていない。
	\end{enumerate}
\end{enumerate}

DailyBuildの実行に関わるスクリプトについては、
\Ref{sec}{DailyBuildScript}を参照のこと。
